方法

名称

说明

clearWarnings

清除所报告的有关此 SQLServerConnection 对象的所有警告。

close

立即释放此 SQLServerConnection 对象的数据库和 JDBC 资源，而非等待它们自动释放。

commit

使自上次提交或回滚以来所做的所有更改成为永久更改，并释放当前由此 SQLServerConnection 对象持有的任何数据库锁。

createBlob

创建不包含任何数据的 java.sql.Blob 对象。

createClob

创建不包含任何数据的 java.sql.Clob 对象。

createNClob

创建不包含任何数据的 java.sql.NClob 对象。

createStatement

创建用于把 SQL 语句发送到数据库的 SQLServerStatement 对象。

createSQLXML

创建不包含任何数据的 java.sql.SQLXML 对象。

getAutoCommit

检索此 SQLServerConnection 对象的当前自动提交模式。

getCatalog

检索此 SQLServerConnection 对象的当前目录名称。

getClientConnectionID 方法 (SQLServerConnection)

无论最新连接尝试成功还是失败，都获取该尝试的连接 ID。

getClientInfo

检索有关 JDBC 驱动程序支持的客户端信息属性信息。

getHoldability

检索使用此 SQLServerConnection 对象创建的 SQLServerResultSet 对象的当前保持能力。

getMetaData

检索包含有关数据库的元数据的 SQLServerDatabaseMetaData 对象，对此数据库而言，此 SQLServerConnection 对象表示一项连接。

getTransactionIsolation

检索此 SQLServerConnection 对象的当前事务隔离级别。

getTypeMap

检索与此 SQLServerConnection 对象关联的 Map 对象。

getWarnings

检索调用此 SQLServerConnection 对象时报告的第一个警告。

isClosed

指示此 SQLServerConnection 对象是否已关闭。

isReadOnly

指示此 SQLServerConnection 对象是否处于只读模式。

isValid

指示此 SQLServerConnection 对象是否已关闭以及是否仍然有效。

nativeSQL

将给定的 SQL 语句转换为数据库服务器的本机 SQL 语法。

prepareCall

创建用于调用数据库存储过程的 SQLServerCallableStatement 对象。

prepareStatement

创建用于将参数化的 SQL 语句发送到数据库的 SQLServerPreparedStatement 对象。

releaseSavepoint

从当前事务中删除指定 SQLServerSavepoint 对象。

rollback

撤消在当前事务中所做的所有更改并释放当前由此 SQLServerConnection 对象持有的任何数据库锁。

setAutoCommit

将此 SQLServerConnection 对象的自动提交模式设置为给定状态。

setCatalog

设置指定的目录名称，以选择在其中使用此 SQLServerConnection 对象的数据库的子空间。

setClientInfo

设置客户端信息属性的值。

setHoldability

将使用此 SQLServerSavepoint 对象创建的 SQLServerResultSet 对象的保持能力更改为给定保持能力。

setReadOnly

将此 SQLServerConnection 对象设置为只读模式，以此作为 JDBC 驱动程序启用数据库优化的提示。

setSavepoint

在当前事务中创建未命名的保存点，并返回代表它的新 SQLServerSavepoint 对象。

setTransactionIsolation

尝试将此 SQLServerConnection 对象的事务隔离级别更改为给定级别。

setTypeMap

安装给定的 TypeMap 对象，作为此 SQLServerConnection 对象的类型映射。
